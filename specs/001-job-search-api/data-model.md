# Data Model: Job Search API

**Branch**: `001-job-search-api` | **Date**: 2026-03-18

## Entities

### Job

Represents a single job listing collected from an external source and stored in the internal DB.

| Field        | Type          | Nullable | Constraints                        | Notes                              |
|--------------|---------------|----------|------------------------------------|------------------------------------|
| id           | Long          | No       | PK, auto-increment (IDENTITY)      | Internal surrogate key             |
| title        | String        | No       | NOT NULL                           | Job posting title                  |
| company      | String        | No       | NOT NULL                           | Company name                       |
| url          | String        | No       | NOT NULL, TEXT                     | Link to original job posting       |
| collectedAt  | LocalDateTime | No       | NOT NULL, updatable = false        | Timestamp when collected from source |

**Kotlin entity**:

```kotlin
@Entity
@Table(name = "jobs")
class Job(
    @Column(nullable = false)
    val title: String,

    @Column(nullable = false)
    val company: String,

    @Column(nullable = false, columnDefinition = "TEXT")
    val url: String,

    @Column(nullable = false, updatable = false)
    val collectedAt: LocalDateTime = LocalDateTime.now(),
) {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    val id: Long = 0
}
```

**Notes**:
- All fields are `val` (immutable after creation) per constitution Principle I.
- `collectedAt` defaults to `LocalDateTime.now()` at object creation.
- No relationships to other entities in this iteration.
- Additional fields (description, tags, salary, location) deferred to AI integration phase.

## DTOs

### JobSearchRequest (request)

```kotlin
data class JobSearchRequest(
    val tags: List<String> = emptyList(),
    val query: String = "",
)
```

**Validation** (enforced in service via `require()`):
- `tags.isNotEmpty() || query.isNotBlank()` — at least one must be provided.

---

### JobResponse (response)

```kotlin
data class JobResponse(
    val id: String,
    val title: String,
    val company: String,
    val match: Int,
    val reason: String,
    val url: String,
)
```

**Notes**:
- `id` is `String` (spec requirement) — converted from `Long` entity id via `.toString()`.
- `match` is always `0` for this iteration (placeholder).
- `reason` is generated by rule-based template in service.

## Reason Generation Logic

The `reason` string is generated in `JobService` based on the request content:

| Condition               | Template                                                              |
|-------------------------|-----------------------------------------------------------------------|
| tags only               | `"선택하신 {tags} 키워드와 관련된 최신 공고입니다."`                  |
| query only              | `"\"{query}\" 관련 최신 공고입니다."`                                |
| tags + query            | `"선택하신 {tags} 키워드와 \"{query}\" 조건에 맞는 최신 공고입니다."` |

Where `{tags}` = `tags.joinToString(", ")`.
